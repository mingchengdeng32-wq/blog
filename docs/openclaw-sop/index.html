OpenClaw 是什么

一句话说明

OpenClaw 是一款开源 AI 智能助手框架，可以把 AI 大模型（如 DeepSeek、Kimi、通义千问等）接入到飞书、企业微信、Telegram 等平台，实现自动化任务执行。

通俗理解

想象一个 24 小时在线的私人助理，它住在你的电脑里，可以帮你：处理邮件、整理文档、查询信息、回答问题、执行自动化任务。你通过浏览器网页或聊天工具跟它对话，它就会帮你干活。

🔒
隐私保障：所有数据保存在你自己的电脑上，不会上传到外部服务器。API Key 以加密形式存储在本地。

安装前准备

硬件与系统要求

项目要求

硬盘空间≥ 1GB 可用

你需要提前准备：阿里云百炼 API Key（免费）

OpenClaw 需要接入一个 AI 大模型才能工作。我们推荐使用阿里云百炼平台——国内最稳定的大模型服务，新用户每个模型赠送 100 万 Tokens 免费额度（有效期 90 天），支持通义千问、DeepSeek、Kimi 等数十个模型。

🎁
免费额度说明：每个模型独立计算 100 万 Token，全部开通可获得数千万 Token。90 天内日常使用完全够用，不会产生任何费用。

第一步：注册阿里云账号并开通百炼

A
打开百炼平台

用浏览器打开：https://bailian.console.aliyun.com

如果没有阿里云账号，先用手机号注册一个（需实名认证）。

B
开通百炼服务

登录后页面会自动弹出服务协议，阅读并同意即可。开通不会产生费用，系统会自动发放免费额度。

如果没有弹出协议，说明你已经开通过了。

第二步：创建 API Key

C
进入 API Key 管理页面

在百炼控制台页面，鼠标移到右上角你的头像上，在下拉菜单中点击 "API-KEY"。

D
创建并复制 Key

① 确认页面右上角地域选择为 华北2（北京）

② 点击 "创建 API Key"

③ 归属空间选 "默认业务空间"，权限选 "全部"，点确定

④ 创建成功后，立刻点"复制"把 Key 保存到一个安全的地方（如备忘录）。Key 格式类似 sk-xxxxxxxxxxxxxxxx

⚠️
安全提醒：API Key 相当于密码，创建后只显示一次。请立即复制保存，不要发给任何人、不要截图分享、不要写在公开文档中。

第三步：开启"免费额度用完即停"（防止意外扣费）

这一步非常重要，可以防止免费额度用完后自动扣费：

① 在百炼控制台左侧菜单，点击 "模型用量"

② 找到 "免费额度" 页签

③ 点击 "一键开启所有模型" 的"免费额度用完即停"开关

💡
开启后，当某个模型的免费额度用完，它会停止服务而不是自动扣费。你可以切换到其他还有额度的模型继续使用。

安装步骤

1
以管理员身份打开 PowerShell

① 按键盘 Win + S（Win 键 = 键盘左下角 Windows 图标键）

② 搜索框输入 PowerShell

③ 在结果中 右键 点击 "Windows PowerShell"

④ 选择 "以管理员身份运行"

⑤ 弹出提示时点击 "是"

💡
PowerShell 是一个蓝色/黑色背景的命令行窗口，这是正常的，不要害怕。
2
允许执行脚本

在 PowerShell 中复制粘贴以下命令，按回车：

PowerShell 
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

如果提示确认，输入 Y 然后按回车。

💡
这一步在做什么？ Windows 默认禁止运行外部脚本。这条命令允许运行经过签名的脚本，仅影响当前用户，是安全的。
3
运行一键安装命令

下面这行命令虽然看起来像乱码，但它就是一句"帮我自动安装 OpenClaw"的意思。复制粘贴到 PowerShell，按回车，然后等着就行：

PowerShell 
iwr -useb https://openclaw.ai/install.ps1 | iex

安装脚本会自动：检测并安装 Node.js → 安装 OpenClaw → 启动配置向导。通常需要 2–5 分钟。

✅
看到绿色文字 "Installation complete!" 表示安装成功。
!
如果上面的命令失败了？

国内网络有时会导致脚本下载失败。请用以下替代方案：

① 手动安装 Node.js：打开浏览器访问 nodejs.org，下载 LTS 版本（绿色按钮），双击安装，一路"下一步"。

② 关闭 PowerShell，重新以管理员身份打开。

③ 依次运行以下命令：

PowerShell 
# 验证 Node.js（应显示 v22 或更高）
node -v

# 设置国内镜像（解决下载慢）
npm config set registry https://registry.npmmirror.com

# 安装 OpenClaw
npm install -g openclaw
4
验证安装成功
PowerShell 
openclaw --version

如果显示版本号（如 2026.3.28），说明安装成功！

4

首次配置向导

启动向导

PowerShell 
openclaw onboard --install-daemon
💡
--install-daemon 的作用：让 OpenClaw 在后台持续运行。关闭终端窗口也不会停止服务。一定要加上这个参数。

向导各步骤操作指引

向导是交互式的。全程使用键盘操作：用 ↑↓ 方向键移动选择，按空格选中/取消，按回车确认。不需要用鼠标。

步骤屏幕显示你的操作
安全警告Continue?选 Yes，回车
选择模式QuickStart / Advanced选 QuickStart，回车
选择模型模型列表选 Skip for now（先跳过，我们后面手动配百炼）
聊天渠道Telegram / WhatsApp / ...选 Skip for now，回车
SkillsConfigure skills?选 No，回车
HooksEnable hooks?选 Skip for now，回车
GatewayInstall / Restart选 Install（或 Restart），回车
🎉
配置完成后，浏览器会自动打开控制面板。但此时还没有接入 AI 模型，继续下面的步骤配置百炼。

配置百炼模型（关键步骤）

向导跳过了模型选择，现在通过 OpenClaw 的 Web 控制面板手动配置百炼：

1
打开 OpenClaw 控制面板的配置页面

浏览器打开 http://127.0.0.1:18789

在左侧菜单点击 配置 → RAW（或 Config → RAW）

2
粘贴百炼配置

把下面这段配置完整复制，粘贴到 RAW JSON 输入框中。这些参数都已经帮你配好了，你只需要把其中的 sk-你的百炼APIKey 改成你自己真实的 Key，其他内容一个字都不要动：

JSON 配置 
{
  "models": {
    "mode": "merge",
    "providers": {
      "bailian": {
        "baseUrl": "https://dashscope.aliyuncs.com/compatible-mode/v1",
        "apiKey": "sk-你的百炼APIKey",
        "api": "openai-completions",
        "models": [
          {
            "id": "qwen3.5-plus",
            "name": "Qwen3.5-Plus",
            "reasoning": false,
            "input": ["text", "image"],
            "cost": { "input": 0, "output": 0 },
            "contextWindow": 1000000,
            "maxTokens": 65536
          },
          {
            "id": "qwen3-coder-next",
            "name": "Qwen3-Coder-Next",
            "reasoning": false,
            "input": ["text"],
            "cost": { "input": 0, "output": 0 },
            "contextWindow": 262144,
            "maxTokens": 65536
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": "bailian/qwen3.5-plus",
      "models": {
        "bailian/qwen3.5-plus": {},
        "bailian/qwen3-coder-next": {}
      }
    }
  }
}
⚠️
一定要替换 API Key：把 sk-你的百炼APIKey 换成你自己的真实 Key（以 sk- 开头的一串字符）。Key 前后不要有空格。
3
保存并重启

① 点击右上角 Save 保存

② 点击 Update 使配置生效

③ 回到聊天界面，发送"你好"测试——如果收到回复，说明配置成功！

（可选）配置 Windows 环境变量

把 API Key 设成系统环境变量后，OpenClaw 可以自动读取，更安全也更方便。虽然上面已经通过配置文件设好了，但设环境变量是业界最佳实践（防止 Key 写在文件中被误传播）：

1
打开环境变量设置

按键盘 Win + Q，在搜索框输入 "编辑系统环境变量"，点击打开

2
新建环境变量

① 在弹出的"系统属性"窗口中，点击 "环境变量" 按钮

② 在 "用户变量" 区域，点击 "新建"

③ 变量名填：DASHSCOPE_API_KEY

④ 变量值填：你的百炼 API Key（以 sk- 开头的那串字符）

⑤ 点击 "确定" 保存

3
验证是否生效

关闭所有 PowerShell 窗口，重新打开一个，运行：

PowerShell 
echo $env:DASHSCOPE_API_KEY

如果屏幕上显示了你的 API Key，说明环境变量设置成功。

💡
为什么要重新打开 PowerShell？ 环境变量修改后，已经打开的窗口不会自动刷新，必须重新打开才能读到新的值。

网络安全说明

🔒
OpenClaw 默认只监听 127.0.0.1（本机回环地址），仅本机可访问。局域网内其他电脑无法连接你的 OpenClaw。
🚫
请勿在未经 IT 部门确认的情况下修改网关绑定地址或关闭沙箱模式。如有需要请联系 IT 同事协助。

常见问题

现象：运行一键安装命令后，PowerShell 显示红色错误或长时间无响应。

解决：使用替代方案——先去 nodejs.org 手动安装 Node.js，再用 npm install -g openclaw 安装。详见第 3 节"如果命令失败了"。

现象：输入 openclaw --version 提示"无法将 openclaw 项识别为 cmdlet"（意思是电脑找不到这个命令）。

解决：① 关闭 PowerShell → ② 重新以管理员身份打开 → ③ 再试。如果仍然不行，运行 npm config get prefix，把输出的路径添加到系统环境变量 PATH（环境变量就像电脑的"通讯录"，告诉它去哪里找程序。不会操作的话请联系 IT 同事协助）。

现象：浏览器访问 http://127.0.0.1:18789 无法加载。

解决：① 运行 openclaw gateway status 检查状态 → ② 如果 stopped，运行 openclaw gateway start → ③ 确认用的是 http:// 不是 https:// → ④ 临时关闭防火墙测试。

现象：发送消息后无回复，或提示认证失败。

解决：① 检查 API Key 是否正确（复制时不要有多余空格） → ② 确认模型账号有余额 → ③ 运行 openclaw configure 重新配置。

现象：安装过程中提示版本不满足要求。

解决：去 nodejs.org 下载最新 LTS 版本重新安装，安装后必须关闭 PowerShell 重新打开。

现象：启动 Gateway 时提示端口被占用。

解决：

PowerShell 
# 查找占用端口的进程
netstat -ano | findstr :18789

# 记下最后一列的 PID 数字，结束该进程
taskkill /PID 替换为PID数字 /F

卸载与重置

完整卸载

PowerShell 
# 第 1 步：OpenClaw 自带卸载
openclaw uninstall

# 第 2 步：卸载 npm 全局包
npm uninstall -g openclaw

# 第 3 步（可选）：删除所有配置和数据
Remove-Item -Recurse -Force "$env:USERPROFILE\.openclaw"
⚠️
第 3 步会永久删除所有 OpenClaw 配置和对话数据，执行前请确认。

重置配置（保留安装）

PowerShell 
openclaw onboard --reset
